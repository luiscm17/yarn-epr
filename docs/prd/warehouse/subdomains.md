# Subdomains — Warehouse Unit

> **Chained from:** `docs/prd/warehouse.md`
> **Version:** 0.1

---

## Transversal structure

All three subdomains share the same stock calculation formula:

```
(Previous Balance + Entries) - Exits = Current Balance
```

Each movement (entry or exit) records: movement number, date, authorized by, received/delivered by, and concept.

---

## 1. Raw Material (MP)

### 1.1 Purpose

Manage the reception of raw material bales (wool, acrylic, blends), register incoming lots/trucks, assign the unique lot code, enrich with order data, and emit to Production.

### 1.2 Operational cycle

```
Supplier → Reception → Lot registration → Code assignment → Enrichment → Emission to Production
```

### 1.3 Reception and lot registration

When bales arrive, the Warehouse Assistant registers the incoming lot:

| Field | Description |
|---|---|
| **Entry number** | Auto-generated by the system |
| **Date** | Reception date |
| **Supplier** | From supplier catalog (e.g. Illampu Textiles) |
| **Invoice / Receipt** | Supplier invoice number |
| **Source** | Typically "Own Resources" |
| **Concept** | Brief description of the entry |
| **Authorized by** | Person authorizing reception |
| **Delivered to** | Warehouse responsible receiving the material |

**Reception items:**

| Field | Description |
|---|---|
| **Quantity (kg)** | Gross weight received |
| **Title** | Yarn title (e.g. 2/18, 2/32, 4/9) |
| **Color / Fiber** | Fiber type or color if applicable |
| **Truck number** | Distributor truck identifier (part of the lot code) |

### 1.4 Unique lot code assignment

The system generates the code `NN-GGGG-NNN`:

- `NN` — Distributor/truck number
- `GGGG` — Management year
- `NNN` — Sequential entry number for the year

This code is the **lot's unique identifier** throughout its entire lifecycle.

### 1.5 Enrichment

Before emission to Production, Warehouse registers order data that will accompany the lot:

| Field | Source |
|---|---|
| **Client** | Order destination |
| **Color** | Requested color (e.g. Rojo 3006, Negro 5000) |
| **Title** | Yarn title |
| **Type N / CH** | Title variant (N fiber, fine CH) |
| **Observations** | Order notes |

### 1.6 Emission to Production

When emitting MP, the system records:

- **Emission date**
- **Receiving supervisor** (shift supervisor)
- **Emitted quantity (kg)**
- **Destination** (production section)

Once emitted, the MP leaves Warehouse's responsibility.

### 1.7 Catalogs

- Suppliers (general, shared across subdomains)
- Titles (2/18, 2/32, 4/9, Stoll, Fantasia, etc.)
- Colors / fibers (free text, grows organically)
- Clients (shared with Finished Product)

### 1.8 Unit of measure

Kilograms (kg).

## 2. Finished Product (PT)

### 2.1 Purpose

Manage reception of finished product from Production, storage (bagged or bulk), and exits via direct sale or transfer to Comercializacion.

### 2.2 Operational cycle

```
Production → PT Reception → Storage (bagged / bulk) → Exit
                                                         ├── Direct sale (full lot)
                                                         └── Comercializacion (units)
```

### 2.3 PT Reception from Production

When Production completes a lot, Warehouse receives the PT and registers:

| Field | Description |
|---|---|
| **Reception number** | Auto-generated |
| **Date** | Reception date |
| **Lot code** | `NN-GGGG-NNN` (inherited from MP) |
| **Description** | Full item description (e.g. "Rojo 3006 2/18") |
| **Quantity (kg)** | Received weight |
| **Value** | Lot monetary value |
| **Category** | From PT catalog |
| **Subcategory** | From PT catalog |
| **Concept** | Lot reception from Production |
| **Authorized by** | Warehouse responsible |
| **Delivered to** | Warehouse assistant |
| **State** | Bagged or Bulk |

### 2.4 Storage states

PT arrives from Production in two formats:

- **Bagged:** ready for direct sale. Standard weight per bag.
- **Bulk:** full lot available for wholesale sale.

Warehouse may reorganize between these states as needed.

### 2.5 Exits

**Direct sale to client (full lot):**

| Field | Description |
|---|---|
| **Exit number** | Auto-generated |
| **Invoice number** | From the invoice booklet |
| **Date** | Exit date |
| **Exit type** | External Exit (fixed) |
| **Destination (Client)** | From client catalog |
| **Authorized by** | Warehouse Chief or Production Chief |
| **Concept** | Exit description |
| **Delivered to** | Transport responsible / client |

**Transfer to Comercializacion (units):**

Same structure but with:
- **Destination:** Comercializacion
- **Exit type:** Transfer to Comercializacion
- Quantity recorded in **units** (not kg)

### 2.6 Returns

For lots returned by clients or Comercializacion:

- Registered as entry with category "Returns"
- Lot code may include return suffix
- Reincorporated to stock with updated value

### 2.7 Catalogs

- Categories and subcategories (hierarchical structure)
- Clients (shared with Raw Material)
- PT supplier (Illampu Textiles, fixed)

### 2.8 Unit of measure

Kilograms (kg) for stock. Units for Comercializacion transfers.

## 3. Supplies

### 3.1 Purpose

Manage all production supplies inventory: dyes, chemicals, packaging, spare parts, lubricants, cones, tubes, and any other consumable used by Production.

### 3.2 Catalog structure

Unlike Raw Material and Finished Product, Supplies does not have a fixed catalog. It is organized by **categories** that can grow over time:

| Level | Example |
|---|---|
| **Category** | Dyehouse, Packaging, Maintenance, Lubricants, Cones & Tubes |
| **Item** | Specific product with its own unit of measure |

Initial categories (subject to expansion):

| Category | Description | Unit |
|---|---|---|
| **Dyehouse** | Dyes, chemical inputs for dyeing process | kg, L |
| **Packaging** | Bags, labels, winding cards, booklets | pieces |
| **Maintenance** | Spare parts, tools | pieces |
| **Lubricants & Oils** | Machine lubricants, industrial oils | L |
| **Cones & Tubes** | Cardboard cones, tubes, tape | pieces |
| **Others** | Any other production supply not fitting above categories | varies |

Each item has its own unit of measure (kg, L, pieces, meters, etc.).

### 3.3 Entry (from supplier)

| Field | Description |
|---|---|
| **Entry number** | Auto-generated |
| **Date** | Entry date |
| **Category** | From Supplies catalog |
| **Supplier** | From supplier catalog |
| **Entry type** | Storage Entry, Return, Sample |
| **Authorized by** | Warehouse or Production responsible |
| **Delivered to** | Warehouse responsible |

### 3.4 Exit (consumption by Production)

| Field | Description |
|---|---|
| **Exit number** | Auto-generated |
| **Date** | Exit date |
| **Category** | From Supplies catalog |
| **Destination** | Production area receiving the supply |
| **Exit type** | Production consumption, Return to supplier |
| **Authorized by** | Warehouse or Production responsible |
| **Delivered to** | Receiving production area |

### 3.5 Catalogs

- Categories (growable, defined by Warehouse and Production)
- Suppliers (shared with Raw Material)
- Items with per-unit pricing

### 3.6 Unit of measure

Varies per item: kg, L, pieces, meters, depending on the supply category.
